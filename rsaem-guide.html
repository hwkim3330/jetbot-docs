알쌤로봇 (RSAEM Robot) 사용자 가이드

RSAEM 로봇은 NVIDIA Jetson Orin Nano를 기반으로 한 교육용 ROS2 로봇 플랫폼입니다. ROS2 Humble, Cartographer SLAM, Nav2 Navigation을 지원하며, 2D LiDAR와 Dynamixel 모터를 탑재하고 있습니다.

문서 정보
  • 문서 번호: RS250512011
  • 문서 버전: RS2505001
  • 작성: 주식회사 젯슨에이아이

주의 사항

1. 충전 규격 준수 (필수!)

반드시 8.4V 어댑터로 충전해야 합니다. (판매 시 동봉)

지정된 규격 어댑터로 충전하지 않는 경우 로봇 배터리 시스템 및 전원 시스템의 손상이 일어날 수 있으며, 이 경우 드물게 오린 나노의 전원 장치가 손상될 수 있습니다.

2. 이동 시 전원 차단

로봇의 전원이 켜진 상태로 박스에 넣어 이동하지 않습니다. (쇼트 위험)

3. Pi 카메라 케이블 관리

Pi 카메라 특성 상 케이블 연결이 느슨해지는 경우 카메라 동작이 되지 않습니다. 연결 상태 유지가 중요하며, 케이블에 손상이 생기면 역시 카메라 동작이 되지 않습니다.

4. USB 과부하 주의

오린 나노의 USB 포트에 유전원 USB를 연결하는 등 너무 많은 USB를 연결하면 문제가 생길 수 있습니다.

5. 전원 모드 설정 권장

배터리 연결하여 사용하는 모빌리티 장비이기 때문에 안정적인 배터리 사용을 위해 오린 나노의 파워모드를 15W에 맞추어 사용하는 것을 권장합니다.

# 15W 모드 설정
sudo nvpmodel -m 0

RSSAEM ROS2 패키지 빌드

패키지 빌드 (소스 전달 후)

# 워크스페이스로 이동 후 빌드
cd rssaem_ws
colcon build

환경변수 설정 (.bashrc)

아래 명령은 최초 1회만 실행합니다.

# bashrc에 환경 설정 추가
echo "source ~/rssaem_ws/install/setup.bash" >> ~/.bashrc
echo "export LIDAR_MODEL=LDS-04" >> ~/.bashrc
echo "export RSSAEM_MODEL=rssaem" >> ~/.bashrc

# 현재 세션에 적용
source ~/.bashrc

로컬 Teleop 테스트

Step 1: RSAEM 로봇 실행

# 터미널 1: RSAEM Bringup
# * export LIDAR_MODEL은 bashrc에 있기 때문에 입력하지 않아도 됩니다
ros2 launch rssaem_bringup rssaem.launch.py

Step 2: Teleop 실행 (다른 터미널)

# 터미널 2: 키보드 텔레오퍼레이션
ros2 run rssaem_teleop teleop_keyboard
종이컵 테스트 필수!

바퀴가 땅에 닿지 않도록 종이컵 위에서 바퀴가 움직이는지 확인해주세요.

Step 3: ROS2 토픽 및 그래프 확인

# 터미널 3: 토픽 목록 확인
ros2 topic list

# 노드 연결 관계 확인
rqt_graph

Step 4: RViz2 확인 (다른 터미널)

# 터미널 4: RViz2 시각화
ros2 launch rssaem_bringup rviz2.launch.py
확인 사항:
  • RViz 상의 노드들과 TF들을 확인해주세요
  • LaserScan 데이터가 표시되는지 확인
  • RobotModel이 올바르게 표시되는지 확인

PC 환경 세팅 (원격 제어용)

로봇 IP 확인

# 로봇에서 실행
ifconfig
중요: 원격 테스트를 위해 로봇의 터미널창을 모두 닫아주십시오 (필수)

이제 PC에서만 명령을 내리게 됩니다.

PC bashrc 수정 (최초 1회)

# bashrc 편집
cd
gedit .bashrc

# 아래 내용 추가 (본인의 ROS_DOMAIN_ID로 수정)
export ROS_DOMAIN_ID=30

# 저장 후 창 닫기

PC 빌드 (소스 복사 후)

# 워크스페이스로 이동 후 빌드
cd rssaem_ws
colcon build

# bashrc에 환경 설정 추가 (최초 1회)
echo "source ~/rssaem_ws/install/setup.bash" >> ~/.bashrc
echo "export LIDAR_MODEL=LDS-04" >> ~/.bashrc
echo "export RSSAEM_MODEL=rssaem" >> ~/.bashrc

source ~/.bashrc

원격 Bringup

Step 1: SSH 접속

# PC에서 로봇으로 SSH 접속
ssh nvidia@{MY_RSSAEM_BOT_IP}

# 예시
ssh nvidia@192.168.100.99

Step 2: 로봇 Bringup 실행

# SSH 접속 후 로봇에서 실행
ros2 launch rssaem_bringup rssaem.launch.py

# 터미널에 에러가 없는지 확인해주세요

Step 3: 토픽 확인 (PC의 다른 터미널)

# PC에서 새 터미널 열고 실행
ros2 topic list

# 노드 연결 관계 확인
rqt_graph

Step 4: 원격 RViz2 확인

# PC에서 RViz2 실행
ros2 launch rssaem_bringup rviz2.rssaem.launch.py
확인 사항:
  • RViz 상의 노드들과 TF들을 확인해주세요
  • 손으로 로봇을 돌려보거나 바닥에서 Teleop으로 이동해보세요

원격 제어

Step 1: SSH로 로봇 Bringup

# PC에서 로봇으로 SSH 접속
ssh nvidia@{MY_RSSAEM_BOT_IP}

# 로봇에서 Bringup 실행
ros2 launch rssaem_bringup rssaem.launch.py

Step 2: Teleop 실행 (PC의 다른 터미널)

# PC에서 teleop 실행
ros2 run rssaem_teleop teleop_keyboard
테스트 순서:
  1. 동작 여부를 확인
  2. 충전 케이블을 뽑고
  3. 바닥에서 전진, 후진, 좌회전, 우회전 등 동작을 확인

테스트 완료 후 모든 창을 닫아주세요.

SLAM 실습

환경 준비:
  • 펜스를 비대칭으로 설치해주세요
  • 로봇 시작 위치와 방향을 기억해주세요 (Nav2에서 사용)

Step 1: 로봇 Bringup

# PC에서 SSH 접속
ssh nvidia@{MY_RSSAEM_BOT_IP}

# 로봇에서 Bringup 실행
ros2 launch rssaem_bringup rssaem.launch.py

Step 2: Cartographer 실행 (다른 터미널)

# PC에서 Cartographer 실행
ros2 launch rssaem_cartographer cartographer.launch.py

Step 3: Teleop 실행 (다른 터미널)

# PC에서 teleop 실행
ros2 run rssaem_teleop teleop_keyboard
매핑 팁:
  • Teleop을 이용하여 맵을 완성합니다
  • 로봇 속도는 0.3 이하로 천천히 진행해야 rf2o odom이 깨끗한 지도를 만드는데 도움이 됩니다
  • 검은 색 테두리로 폐쇄 공간이 될 수 있도록 해주십시오
  • 같은 장소를 다시 방문하면 루프 클로징이 됩니다

Step 4: 맵 저장 (다른 터미널)

# PC에서 맵 저장
ros2 run nav2_map_server map_saver_cli -f ~/map

# 생성되는 파일:
# - ~/map.pgm (이미지 파일)
# - ~/map.yaml (메타데이터)
저장 완료: 저장 후 모든 창을 닫아주세요.

Nav2 실습

중요: SLAM 때 사용한 처음 위치와 방향에 로봇을 위치시켜주세요!

Step 1: 로봇 Bringup

# PC에서 SSH 접속
ssh nvidia@{MY_RSSAEM_BOT_IP}

# 로봇에서 Bringup 실행
ros2 launch rssaem_bringup rssaem.launch.py

Step 2: Nav2 실행 (다른 터미널)

# PC에서 Navigation2 실행 (저장한 맵 사용)
ros2 launch rssaem_navigation2 navigation2.launch.py map:=$HOME/map.yaml

Step 3: 목표점 설정

RViz2가 자동으로 실행됩니다. 다음 기능을 사용할 수 있습니다:

RViz2 Navigation Panel 기능:
  • 2D Pose Estimate: 초기 위치 설정 (로봇 위치가 맞지 않을 때)
  • Nav2 Goal: 목표 위치 설정 (클릭 후 드래그로 방향 지정)
  • Waypoint / Nav Through Poses Mode: 다중 경유점 설정
Navigation 2 Panel 항목 설명
Navigation active / inactive 상태 표시
Localization active / inactive 상태 표시
Feedback unknown / reached / navigating 상태
ETA 예상 도착 시간
Distance remaining 남은 거리 (m)
Time taken 소요 시간
Recoveries 복구 동작 횟수
성공!

Feedback이 "reached"로 표시되면 목표 지점에 도착한 것입니다.
